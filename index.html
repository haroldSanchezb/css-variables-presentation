CSSConf Review

Before CSS variables

A lot of repeated values

CSS variables

Are entities defined by CSS

Contain specific values to be reused

Are set using custom property notation 

--my-var: value;

Are accessed using the var() function 

color: var(--main-color);

Basic usage

Declaring a variable:


              element {
                --main-bg-color: green;
              }

Using the variable:


                element {
                  background-color: var(--main-bg-color);
                }

Global variables

Reduce the need for repetition

Declare variables on :root pseudo-class

Using :root pseudo-class:


              :root {
                --body-font-size: 14px;
              }

Fallback values

If the variable is not defined yet

Receives two arguments

Basic usage:


              .two {
                color: var(--my-var, red);
                /* Red if --my-var is not defined */
              }

Multiple variables:


              .three {
                background-color: var(--my-var, var(--my-background, pink));
                /* pink if my-var and --my-background are not defined */
              }

Bad usage:


              .three {
                background-color: var(--my-var, --my-background, pink);
                /* Invalid: "--my-background, pink" */
              }

Invalid variables

When the browser encounters an invalid var() substitution

The initial or inherited value of the property is used

Media Queries

God save the :root

Just setup the :root pseudo-class in your media query

Usage:


              :root {
                --root-color: red;
              }

              .six {
                background-color: var(--root-color);
              }

              @media (orientation: portrait) and (pointer: coarse) {
                :root {
                  --root-color: green;
                }
              }

Final Demo

So...

You can use them without the need of a preprocessor

They cascade. You can set a variable inside any selector to set or override its current value

When their values change (e.g. media query or other state), the browser repaints as needed

You can access and manipulate them in JavaScript
